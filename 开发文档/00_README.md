# AIGC 情景与剧情生成项目开发文档

> 项目暂定名：**StoryWorld**  
> 项目定位：**基于世界状态演化的情景—剧情联合生成系统**

本文件夹是一套面向开发阶段使用的 Markdown 文档。第一版目标不是从零训练大模型，而是先搭建一个能跑通、能迭代、能展示的 AIGC 系统框架。

---

## 1. 项目一句话说明

StoryWorld 希望根据用户输入的世界观、人物设定和剧情种子，自动生成剧情事件，维护世界状态，并进一步生成场景卡和图像 Prompt。

核心链路：

```text
用户输入设定
  ↓
解析为结构化故事设定
  ↓
初始化世界状态 World State
  ↓
生成剧情事件 Event
  ↓
更新世界状态
  ↓
生成场景卡 Scene Card
  ↓
生成图像 Prompt
  ↓
接入图像生成模型 / 游戏素材生成流程
```

## 3. 第一版开发原则

1. **先系统，后模型**  
   不从零训练大模型，先用现成 LLM 完成生成能力，自己重点设计世界状态、状态更新和场景卡。

2. **先文本，后图像**  
   第一步先跑通纯文本流程，等剧情状态链路稳定后，再接图像生成模型。

3. **先 MVP，后高级功能**  
   第一版只做 3～5 个事件的剧情推进，不做复杂交互式游戏、不做强化学习、不做真实具身智能。

4. **所有中间结果结构化**  
   每个模块尽量输出 JSON，便于调试、维护、评估和后续微调。

## 4. 推荐第一周目标

第一周只完成命令行版本：

```text
输入一句故事设定
  ↓
生成初始世界状态 JSON
  ↓
生成第 1 个剧情事件
  ↓
更新世界状态
  ↓
生成第 1 张场景卡
```

完成标准：

- 能在终端运行。
- 能保存生成结果到 `outputs/`。
- 能看到“世界状态被事件改变了”。
- 不需要前端、不需要出图、不需要训练模型。
