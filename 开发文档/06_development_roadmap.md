# 06 开发路线图

## 1. 总体路线

本项目建议按 8 周开发节奏推进。

```text
第 1 周：纯文本单轮原型
第 2 周：多轮剧情推进
第 3 周：场景卡与 Prompt 生成
第 4 周：结果导出与文档整理
第 5 周：前端 Demo
第 6 周：图像生成接入
第 7 周：一致性评估与优化
第 8 周：作品集包装
```

如果时间紧张，前 4 周即可完成一个可展示的文本版 MVP。

---

## 2. 第 1 周：纯文本单轮原型

### 目标

跑通最小链路：

```text
用户输入 → StorySetting → WorldState → EventCard → UpdatedWorldState → SceneCard
```

### 任务

- 建立项目目录。
- 写 `LLMClient` 的假实现。
- 写 `InputParser`。
- 写 `WorldInitializer`。
- 写 `PlotEngine`。
- 写 `StateUpdater`。
- 写 `SceneGenerator`。
- 在 `app.py` 中串起来。

### 完成标准

运行：

```bash
python app.py
```

能够输出：

- 初始故事设定
- 初始世界状态
- 第一个事件
- 更新后的世界状态
- 第一个场景卡

---

## 3. 第 2 周：多轮剧情推进

### 目标

系统可以连续生成 3～5 个事件，并维护状态历史。

### 任务

- 给 `WorldStateManager` 加 `history`。
- 给每个事件增加 `step`。
- 给每个状态增加 `state_id`。
- 让 `PlotEngine` 根据剧情阶段生成不同类型事件。
- 保存每一轮结果。

### 完成标准

输出文件包括：

```text
world_states.json
event_cards.json
scene_cards.json
```

人工阅读时能看出：

- 事件之间有因果关系。
- 人物情绪有变化。
- 线索逐步增加。
- 冲突逐渐升级。

---

## 4. 第 3 周：场景卡与图像 Prompt

### 目标

把每个剧情事件转换为可视化场景，并生成图像 Prompt。

### 任务

- 完善 `SceneCard` 字段。
- 实现 `ImagePromptGenerator`。
- 支持中文 Prompt 和英文 Prompt。
- 加入视觉风格字段。
- 支持固定风格：像素风 RPG CG。

---

## 5. 第 4 周：导出与文档整理

### 目标

让系统输出适合人类阅读和作品集展示的文件。

### 任务

- 实现 `OutputExporter`。
- 导出 `storyboard.md`。
- 导出完整 JSON。
- 整理 README。
- 添加示例运行结果。

---

## 6. 第 5 周：前端 Demo

### 目标

做一个简单网页界面，方便展示。

推荐技术：

- Gradio：更适合 AI Demo。
- Streamlit：更适合数据和可视化展示。

页面设计：

```text
左侧：用户输入故事设定
中间：剧情事件与世界状态
右侧：场景卡与图像 Prompt
```

---

## 7. 第 6 周：接入图像生成

### 目标

根据图像 Prompt 生成情景图。

可选方案：

1. 调用在线图像生成 API。
2. 使用本地 Stable Diffusion WebUI。
3. 使用 Hugging Face Diffusers。
4. 暂时只生成 Prompt，手动复制到图像工具里。

推荐第一步：

```text
系统自动生成 Prompt → 手动复制到图像生成工具 → 保存图片路径
```

等流程稳定后再自动化。

---

## 8. 第 7 周：一致性评估与优化

### 目标

解决前后矛盾、剧情跳跃、场景不清楚等问题。

### 任务

- 增加一致性检查器。
- 记录常见错误。
- 修改 Prompt。
- 增加测试案例。
- 对比不同 Prompt 版本效果。

---

## 9. 第 8 周：作品集包装

### 目标

把项目包装成可以展示的作品。

### 任务

- 写项目介绍。
- 截图 Demo 页面。
- 选 1～2 个最佳生成案例。
- 整理 GitHub README。
- 写技术亮点说明。
- 制作展示视频或 PPT。

技术亮点可以写：

- 使用结构化世界状态维护剧情一致性。
- 使用事件卡驱动状态更新。
- 使用场景卡连接文本生成和图像生成。
- 面向 RPG / 视觉小说场景的 AIGC 工作流。
- 具备向世界模型和交互式叙事扩展的潜力。
